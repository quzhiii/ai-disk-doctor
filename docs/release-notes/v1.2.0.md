# Windows AI Space Manager v1.2.0

## Summary

This release packages the completed Phase 7 workflow for safer AI-era disk diagnostics. It adds lightweight large-file discovery, broader regenerable development artifact coverage, a consistent structured JSON error contract, and cross-platform rule path expansion for key AI tooling roots.

The default posture remains conservative: scan and explain first, keep cleanup dry-run by default, and use quarantine for any real movement. The new discovery features do not add automatic cleanup behavior.

## Included Workflows

- `scan --large-files --min-size 500MB`: discover the largest files and directories under a root path without classification or cleanup suggestions.
- Developer artifact rule coverage: detect common regenerable paths such as `node_modules`, Rust `target/`, Gradle caches, Python `__pycache__`, `dist/`, `.next`, and `.turbo`.
- Structured JSON errors: when `--json` is requested and a command fails, emit one parseable error object to stderr and keep stdout empty.
- Cross-platform rule paths: expand Unix `~/` home paths and Windows `%VAR%` tokens for Ollama, Hugging Face, and Docker rules.
- Path safety refinements: unresolved environment-backed rule paths are skipped instead of producing bogus findings, and equivalent expanded paths are deduplicated before reporting.

## Safety Boundaries

- `scan --large-files` is read-only and never generates cleanup plans by itself.
- Structured JSON errors only change failure serialization; successful JSON reports still go to stdout.
- Cross-platform path support does not add automatic deletion or platform-specific cleanup execution.
- Unresolved `~/` or `%VAR%` paths are skipped rather than guessed.
- Existing cleanup safety boundaries remain unchanged: destructive actions still require explicit `--yes` and quarantine-oriented workflows.

## Test Plan

Run these from the repository root or from `aidisk` as noted:

```powershell
cd aidisk
cargo test
..\scripts\release-smoke.ps1
```

The smoke script exercises community rules, planning, non-destructive clean preview, doctor output, explicit diff rendering, and the new `scan --large-files --min-size 500MB --json` workflow.

## Known Limits

- `scan --large-files` is intentionally discovery-only and does not replace category-aware `scan` or `plan` workflows.
- Cross-platform rules currently focus on path expansion and rule coverage; they do not add platform-specific cleanup commands.
- `doctor` remains Windows-oriented in public positioning even though some rule paths now expand cross-platform.
- Human-readable `--min-size` parsing supports whole-number `B/KB/MB/GB/TB` suffixes only.
