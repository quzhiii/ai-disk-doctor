# Windows AI Space Manager v1.6.0

v1.6.0 adds AI-aware disk diagnostics and a visual dashboard. The doctor system gains a new `--ai-footprint` topic that aggregates all AI tool findings into a single report. A new `visualize --html` command generates an interactive Swiss Style HTML dashboard with bilingual support, category filtering, and a safe reclaim checklist. Rule coverage expands to 25 rules with cross-platform support for 2025-2026 AI tools.

## What Changed

- Added `aidisk visualize --html` to generate a self-contained interactive HTML dashboard with:
  - Bilingual Chinese/English with browser auto-detection and toggle
  - Category treemap with click-to-filter
  - Tool breakdown with expandable details
  - KPI tower with hover tooltips
  - Safe reclaim checklist with select-all and size summary
  - Responsive design for mobile
- Added `aidisk doctor --ai-footprint` to aggregate all AI-related findings across 10 categories.
- Added 5 new AI tool rules: GPU runners (LM Studio, llama.cpp), coding agents (Claude Code, Codex CLI, Gemini CLI), MCP servers, next-gen AI IDEs (Roo Code, Codeium), and CUDA/cuDNN runtime environments.
- Added model file format detection (`.gguf`, `.safetensors`, `.onnx`, `.mlx`) with `risk: safe`.
- Upgraded 6 existing rules to cross-platform format (`platforms: [windows, linux, macos]`).
- Fixed multi-platform YAML rule parsing to handle both `platform:` string and `platforms:` array formats.
- Expanded `release_artifacts` test suite from 26 to 29 tests.

## Test Plan

- From `aidisk/`: `cargo test --test release_artifacts`
- From `aidisk/`: `cargo test --all`
- `aidisk visualize --html --output test.html` generates valid interactive HTML
- `aidisk doctor --ai-footprint --markdown` aggregates findings across all AI categories

## Safety Boundaries

- `visualize --html` generates read-only HTML reports. No files are modified.
- `doctor --ai-footprint` is a read-only diagnostic. No cleanup is performed.
- AI rule expansions are additive only. No existing risk levels changed.
- Cross-platform rule parsing preserves backward compatibility with existing flat-path YAML format.

## Known Limits

- Dashboard data is sourced from scan snapshots in `.aidisk/reports`; running `aidisk scan` first ensures current data.
- Dashboard file size includes embedded scan data; large scans (>10MB) produce larger HTML files.
- CUDA/cuDNN paths are best-effort detection; custom installations may not be found.
- Model file glob patterns search recursively under user home; large drives may take several seconds.
