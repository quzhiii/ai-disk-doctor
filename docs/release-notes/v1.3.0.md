# Windows AI Space Manager v1.3.0

## Summary

This release packages Phase 9 Local Scheduled Governance. It adds growth anomaly detection on top of scan snapshots and a local governance workflow that can be registered with Windows Task Scheduler.

The default posture remains conservative: the governance workflow scans, compares, reports, and optionally delivers a generic webhook payload. It does not perform cleanup.

## Included Workflows

- `aidisk anomaly --latest`: detect growth anomalies from the newest two scan snapshots.
- `aidisk anomaly --before <FILE> --after <FILE>`: compare explicit snapshots with absolute + relative growth thresholds.
- `run-governance.ps1`: run scan, anomaly, and local report artifact generation in one local governance cycle.
- `governance-event.json`: stable event payload for `anomaly_found`, `pending_history`, and `no_anomaly`.
- Message-friendly fields: `headline`, `summary_markdown`, `top_anomaly_path`, and `top_anomaly_growth_bytes`.
- Generic webhook delivery for governance events, with `webhook-failure.json` capturing failed delivery context.
- Windows Task Scheduler helpers: `register-governance-task.ps1`, `show-governance-task.ps1`, `unregister-governance-task.ps1`, and `test-run-governance-task.ps1`.
- `Start-ScheduledTask` support through `test-run-governance-task.ps1` for immediate test runs of an existing registered task.

## Safety Boundaries

- Local Scheduled Governance is read-first and report-first; it does not perform cleanup.
- The scheduler helpers register, inspect, trigger, or unregister tasks only; they do not delete files or run `clean --yes`.
- Webhook delivery uses the generic governance event contract and does not bind the release to a single IM platform.
- Failed webhook delivery keeps local artifacts and writes `webhook-failure.json` for follow-up.
- The first release does not add a daemon, background service, or automatic cleanup policy.

## Test Plan

Run these from the repository root or from `aidisk` as noted:

```powershell
cd aidisk
cargo test
..\scripts\release-smoke.ps1
```

The smoke script exercises community rules, large-file discovery, planning, non-destructive clean preview, doctor output, explicit diff rendering, and explicit `anomaly --before` rendering.

## Known Limits

- Local governance currently targets Windows Task Scheduler; cron, launchd, and systemd timer adapters are future work.
- The notifier surface is generic webhook only; concrete WeChat / WeCom / Feishu / Slack / Telegram / Discord / email adapters are future work.
- `anomaly --latest` needs at least two scan snapshots before it can produce anomaly results.
- Governance artifacts are local files by default and must be consumed by external automation if centralized reporting is needed.
