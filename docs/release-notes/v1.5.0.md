# Windows AI Space Manager v1.5.0

v1.5.0 packages Cross-Platform Governance with Feishu notifier delivery and governance reliability enhancements. It extends v1.4.0's scheduler-first governance loop with a concrete Feishu adapter, event deduplication for idempotency, configurable notify retry, and a comprehensive cross-platform governance user manual.

## What Changed

- Added Notifier Adapter Foundation with `scripts/governance/send-governance-event.sh` dispatching through `local-file`, generic webhook, or Feishu.
- Added Feishu delivery via `scripts/governance/notifiers/feishu.sh` using `FEISHU_WEBHOOK_URL` environment variable.
- Added `scripts/governance/dedup-governance-event.sh` for governance event idempotency via event hash deduplication.
- Added `scripts/governance/retry-governance-notify.sh` with configurable `--max-retries` (default 3) and `--retry-delay` (default 60s) for notifier reliability.
- Integrated dedup and retry into the Unix governance flow via `run-governance.sh`.
- Added `scripts/governance/templates/feishu-governance.tmpl` for customizable Feishu message templates.
- Added `docs/governance-manual.md` covering prerequisites, quick start for Windows Task Scheduler, cron, launchd, and systemd timer, plus dedup, retry, and troubleshooting.
- Added `docs/cross-platform-verification.md` manual verification checklist for all platforms.
- Extended GitHub CI to Ubuntu and macOS runners alongside Windows.
- Preserved the stable `governance-event.json` contract and no-cleanup, no-daemon governance boundary.

## Test Plan

- From `aidisk/`: `cargo test --test release_artifacts`
- From `aidisk/`: `cargo test --all`
- GitHub CI runs Rust tests on Windows (2025), Ubuntu (latest), and macOS (latest).
- Static release artifact coverage verifies all governance scripts, scheduler adapters, notifier adapter foundation, dedup, retry, templates, and docs.

## Safety Boundaries

- Cross-Platform Governance does not perform cleanup and uses no background daemon.
- Feishu delivery uses `FEISHU_WEBHOOK_URL` from the environment only; the resolved webhook URL is never written into failure artifacts.
- Generic webhook remains the delivery boundary; concrete notifier adapter expansion beyond Feishu is reserved for later.
- `run-governance.sh` requires `bash`, `jq`, `curl` for delivery, and `cargo` for local governance runs.

## Known Limits

- cron does not expose first-class last/next run metadata like Windows Task Scheduler or systemd timer.
- launchd and systemd timer scripts are static contract-tested in this repository; full native scheduler execution should be verified on macOS and Linux hosts.
- Concrete IM notifier adapter support such as Slack, WeChat, DingTalk, or email is not included in v1.5.0.
- Retry uses linear delay (not exponential backoff) and does not include jitter.
- Dedup hash is based on key governance event fields; if the same anomaly is detected with a slightly different generated_at timestamp, it will not be deduplicated.
