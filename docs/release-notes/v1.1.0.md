# Windows AI Space Manager v1.1.0

## Summary

This release completes the Doctor V2 diagnostics work for AI-era storage growth. It keeps the existing conservative workflow intact while making `aidisk doctor` more useful for real AI tooling directories, recent growth analysis, and optional tool probes.

The public doctor topic flags remain unchanged: `--docker`, `--wsl`, `--ollama`, `--playwright`, `--huggingface`, and `--agents`. Default doctor runs and explicit flags now share the same internal topic registry.

## Included Workflows

- `doctor --agents`: diagnose Claude, Codex, Gemini, opencode, AI IDE/CLI state, runtime caches, installers, installed application roots, and generated test artifacts.
- Bounded child breakdowns: active findings show top direct children by size so large AI roots are explainable without deleting anything.
- Data-driven recommendations: advice changes for missing paths, tiny placeholders, large roots, and cache-like child directories.
- `doctor --probe-tools`: opt-in external probes for Docker, WSL, and Ollama using explicit probe status reporting.
- `doctor --latest`: append growth context from the newest two scan snapshots, with `--reports-dir` for custom snapshot locations.
- `DoctorTopicSpec` topic registry: centralizes built-in topic names, default enablement, matching logic, recommendations, and optional probe metadata.

## Safety Boundaries

- Doctor remains read-only; it never moves, deletes, compacts, or quarantines files.
- External probes only run when `--probe-tools` is explicitly provided.
- Probe failures are reported as status/output and do not fail the doctor report.
- `doctor --latest` only reads scan snapshot files and preserves the structured JSON shape.
- The topic registry is internal code metadata only; this release does not add a public `--topic` flag or YAML topic metadata format.

## Test Plan

Run these from the repository root or from `aidisk` as noted:

```powershell
cd aidisk
cargo test
cargo run -- doctor --agents --markdown
cargo run -- doctor --docker --probe-tools --markdown
cargo run -- doctor --agents --latest --json
```

`doctor --latest` requires at least two scan snapshots in `.aidisk/reports` unless `--reports-dir` points to a directory with enough snapshots.

## Known Limits

- Topic metadata is still code-side; external YAML topic metadata is future work.
- There is no public `--topic <name>` selector yet.
- External probes depend on local tool availability and may report `not-available`, `failed`, or `error`.
- Doctor recommendations are diagnostic guidance only and should be followed by `plan`, `clean --dry-run`, official tool commands, or manual review.
